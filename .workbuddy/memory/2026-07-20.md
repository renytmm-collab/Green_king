## 对比原版的差距分析（用户问"还缺什么"）
原版是单人闯关塔防（15关递进、升级点数/工人科技树、贩卖建筑、本地存档、教学关）；我们做的是双人实时对战重制版。差距分两层：
- **双人范畴内仍缺的策略深度**：升级/科技树（数值全固定）、多种兵种（仅1种自爆兵）、贩卖/拆除建筑（无退款）、塔特殊能力（仅单一伤害+射程）、工人/作坊经济（我们自动产金）。
- **原版单人/元游戏骨架（另一种模式）**：单人战役+AI敌人、15关进度/解锁、本地存档、新手教学、音效音乐。
下一步待用户定：优先补"双人深度"还是新开"单人战役+AI"。

## 落地 ④地图随机金币拾取（中立、可抢，用户拍板）
用户澄清：本地成就对 PvP 无意义 → 跳过不做。矿产金币仍**自动**入账不变；原版另有**地图上随机出现的金币**，需**点击回收**、**超时消失**、且**双方都能点（谁先点谁得）**。据此实现中立随机金币系统：
- `server/config.js` 新增 `PICKUP_CONFIG`（spawnIntervalMs 6500 / value 30 / lifetimeMs 6000 / maxActive 3）并导出。
- `server.js`：
  - room 新增 `coins:Map` / `coinElapsedMs` / `nextCoinId`；`publicState` 输出 `coins` 数组（id/x/y/value/remainingMs/lifetimeMs）；`resetRoom` 清空。
  - 新增 `cellCenterPoint`、`randomWalkableCell`（扫 grass && !blocked && !buildingId && !obstacleId 且未被现有金币占用）、`spawnCoin`（受 maxActive 限制）、`collectCoin(socket,coinId)`（校验→加金→pushEffect `coin_collect`→broadcastState）。
  - `simulationStep`：矿收益循环后加金币生成（按 spawnIntervalMs）与到期删除循环。
  - `handleMessage` 加 `collect_coin` 分支；导出 `PICKUP_CONFIG/spawnCoin/collectCoin`。
- 前端：`audio.js` 加 `coin` 音效；`renderer.js` 加 `coinAtPoint`（命中半径 cell*0.6）与 `drawCoins`（¥ 金币上下浮动 + 倒计时环 绿→红）；`app.js` pointerdown 最前置金币命中→发 `collect_coin`，`handleGameEvents` 处理 `coin_collect`（爆炸粒子+音效），动画循环 gate 纳入 `state.coins`。
