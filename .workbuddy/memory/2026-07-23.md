# 2026-07-23 工作记录

## ⑨ 无矿山玩家也按一个矿山速度获得金币（基础经济）
需求：即使一个玩家没有矿山，金币也应以一个矿山的速度逐渐增长。
改动 `server.js` 的 `applyMineIncome`：原来按矿建筑逐栋给 owner 加钱（0 矿则 0 收入）；改为按玩家统计 `mineCount`，每人收入 = `Math.max(1, mineCount[owner]) * effectiveMineIncome(room, owner)`。即：0 矿 → 1 矿速度；1 矿 → 1 矿速度（与原行为一致，不翻倍）；N 矿 → N 矿速度。返回值为获得收入的玩家数（原返回值语义为矿数，已被测试当作 >0 布尔使用，且"双玩家均获益"恰好仍=2，老测试仍通过）。空房间（players.size===0）仍返回 0。
`test/server.test.js` 新增 2 项：①"无矿玩家按 1 矿速度"；②"0 矿房间双方仍获基础收入"。
验证：`node --check server.js` 通过；`npm test` 85/85 全绿。

## ⑨-补 保底收入改为固定速度，与矿山逻辑彻底分开
用户反馈：`Math.max(1, mineCount) * effectiveMineIncome` 里的保底用了 `effectiveMineIncome`（含矿山升级加成），导致保底速度会随升级变化。
修正 `server.js` 的 `applyMineIncome`：把保底抽成**固定常量** `baseline = GAME_CONFIG.mineIncome`（一个矿山的基础速度，永不被升级影响），与矿山逻辑完全分开；每人收入 = `Math.max(baseline, mineCount[owner] * effectiveMineIncome(room, owner))`。即：0 矿（无论是否买了矿山升级）→ 固定 `mineIncome`；N 矿 → N×effectiveMineIncome（升级仍生效）。游戏平衡不变（1 矿仍为 1 矿速度），仅修复"0 矿保底也跟着升级放大"的 bug。
`test/server.test.js` 新增 1 项：「买了矿山升级但无矿的玩家，保底仍为裸 `GAME_CONFIG.mineIncome`」。
验证：`node --check server.js` 通过；`npm test` **86/86 全绿**。

## ⑧ 「退出对局」改为「返回大厅」，但保留房间（不退出房间）
用户选择语义：**保留房间可重连** —— 点「返回大厅」只让玩家回到大厅视图，房间与对局在服务器继续、对手不受影响，可随时「返回对局」。

改动：
- `server.js`：删除原 `exitMatch`（会解散房间）。新增 `returnToLobby(socket)`（标记 `player.inLobby=true`、房间保留、给对手发 `opponent_in_lobby`）与 `returnToBoard(socket)`（清 `inLobby`、给对手发 `opponent_returned`）。dispatch 新增 `return_to_lobby` / `return_to_board` / `leave_room`；`leave_room` 复用 `leaveRoom`（彻底退出房间的逃生口）。
- `public/app.js`：新增 `inLobby` 标志；`exitConfirm` 改为发 `return_to_lobby` + `enterLobbyView()`（软返回，保留 playerId/state）；新增 `lobby-return`→`returnToBoard()`、`lobby-exit`→发 `leave_room`+`returnToLobby()`（全量重置）；新增 `enterLobbyView`/`returnToBoard`/`updateLobbyStatus`；`onMessage` 处理 `opponent_in_lobby`/`opponent_returned`；game_state 中若 `inLobby` 调用 `updateLobbyStatus`。
- `public/index.html`：按钮「退出对局」→「返回大厅」；`#exit-overlay` 文案改为「返回大厅？房间保留…」；新增 `#lobby-overlay`（已返回大厅 + 房间号 + 状态 + 「返回对局」/「彻底退出房间」）。
- `public/style.css`：新增 `.ghost-btn`、`.lobby-status`；`.overlay` 已是全屏 z-50 遮罩可拦截棋盘交互。
- `test/server.test.js`：原 `exitMatch` 两项测试改为 `returnToLobby` 测试（房间保留、对手收到 `opponent_in_lobby`、未收到 `room_closed`；无房间时为 no-op）。导入改为 `returnToLobby, returnToBoard`。

验证：
- `node --check` server.js/renderer.js/app.js/config.js 全通过。
- `npm test`：83/83 全绿。
- 端口 3100 启动 `node server.js`：index.html/app.js/style.css/官方塔贴图均 200。
- 真实 WebSocket 端到端脚本：return_to_lobby→对手收到 `opponent_in_lobby`；return_to_board→`opponent_returned`；leave_room→`opponent_left`。全部 OK（临时脚本已删除）。
