# 原版机制纠偏实施报告

## 目标与边界

目标游戏是 4399《绿色王国之战 / The Green Kingdom》。公开攻略能够确认“通过设施扩大领地”“只在己方底盘建造”“兵营路线可指定最多三个目标”和 `Cancel` 等规则；用户提供的目标画面进一步校正了关键表现：矿山自动入账、路线吸附方格并由水平/垂直线段与 90° 转角组成、建筑底盘能进入并覆盖对方领土。

当前项目定位为局域网双人对战，不复制原版关卡、单人 AI、升级或成就。经用户明确授权，本地版本定向接入原版绿色/紫色建筑、树石与士兵行军 PNG；出兵、移动、战斗、建筑摧毁和胜负闭环仍由本项目服务端实现。

## 地图与底盘

地图继续使用 9×18 隐藏逻辑单元，但关卡模板只定义可通行草地 `.` 与悬崖 `#`，不再预先把整张岛屿分给双方。

开局时，每座城堡以半径 1 铺设最多 3×3 的初始底盘。之后每座完成的建筑以相同半径扩张：

- 中立草地变为建筑所属玩家的底盘；
- 对方没有建筑占用的底盘也会被覆盖；
- 对方建筑或城堡所在格保持原归属，不会被隔空吞掉；
- 悬崖不参与扩张；
- 障碍所在格可以被底盘覆盖，但在 `Clear` 前仍不可建造。

这形成了“在边缘建造—铺开八邻格—继续向前建造”的推进链，并允许双方在前线争夺空底盘。

## 矿山自动收入

删除 `coins` 房间状态、金币对象、`collect_coin` 消息、点击命中和金币渲染。统一服务端循环每 2 秒遍历矿山，并直接向矿山所属玩家增加 10 金币。余额变更后广播完整状态。

客户端既不负责计时，也不能提交金币或结算结果。

## 服务端正交网格路线

客户端创建路线时发送：

```text
barracksId / targetBuildingIds / points
```

服务端验证兵营归属、1 至 3 个唯一敌方目标，然后生成：

```text
兵营中心 -> 若干折点 -> 目标 1 中心 -> 若干折点 -> 后续目标中心
```

客户端持续采样拖拽轨迹。服务端验证有限坐标、点数、首尾和目标顺序，再把每个采样点吸附到方格中心；当一次采样同时跨行和跨列时插入一个 L 形转角，之后删除同方向上的冗余点。最终路线最多 24 个折点，且每一段必须水平或垂直，并检查悬崖、树木、岩石和非目标建筑。

因此障碍不能被穿透，但可以通过 90° 转折绕开。客户端伪造越界点、超长轨迹或过多折点会被拒绝。兵营已有正式路线时，新的 `create_route` 会被拒绝；玩家必须先选中兵营并执行 `Cancel`，再从该兵营重新拖画路线。取消与重新创建均由服务端验证，新路线使用新的路线 ID。

客户端预览使用相同的格心吸附和正交转换，画面表现为截图中的亮绿色虚线、水平/垂直线段和直角转弯。服务器返回的正式折线才是士兵移动依据。

服务端随单位状态同步当前左右朝向，因此一个时间步跨过折返点时也不会倒走。客户端使用单一动画帧循环播放行军图集，并将建筑、障碍和士兵按贴图底部纵坐标排序；生命条独立绘制在素材顶部，避免跨格素材互相错误覆盖。

## 服务端战斗循环

服务端以 100ms 固定时间步推进房间。兵营在存在正式路线时每 2.4 秒生成士兵；士兵按目标数组移动，优先与近距离敌军交战，否则攻击当前建筑目标。防御塔自动选择攻击范围内最近的敌军。

士兵、塔和建筑伤害均在服务端计算。单位互相攻击采用同一时间步汇总伤害，避免遍历顺序导致一方先手删除另一方。建筑或士兵生命值归零后从权威状态移除。

建筑是底盘来源，并保存建造顺序。建筑摧毁时，服务端清空动态归属，再按所有存活建筑重新投射底盘，因此失去支撑的前线土地会回退。路线的最终目标被摧毁时，仅取消受影响的正式路线，并立即清除所有以该建筑为最终目标的在途士兵；其他路线的士兵不受影响。被摧毁建筑只是多目标路线的中间目标时，不会误清最终目标仍然存活的在途士兵。玩家主动 `Cancel` 时，同样清除该路线已经派出的士兵，然后才能重新规划路线。

任一城堡生命值归零后房间进入 `finished`，所有建造与战斗推进停止并记录胜者；同一时间步双方城堡同时归零时记录 `draw`。

## WebSocket 状态与消息

- `build`：`row/column/buildingType`
- `clear_obstacle`：`obstacleId`
- `create_route`：`barracksId/targetBuildingIds`
- `cancel_route`：`barracksId`
- `game_state`：`ready/rules/players/map/buildings/obstacles/routes`
- `action_rejected`：`action/reason`

## 自动验证

最终命令：`npm test`

- 测试数量与通过情况以当前 `npm test` 输出为准。

测试覆盖初始底盘、建筑扩张与回退、障碍解锁、矿山结算、拖拽点验证、格心吸附、正交折点、绕过前线障碍、非法穿越、多目标检查点、兵营先取消再重画、按路线取消在途士兵、最终目标摧毁时按路线清兵、中间目标摧毁时不误清、士兵按折点移动与折返朝向、同时近战、塔攻击、城堡胜负、平局、原版 PNG 块校验与图集尺寸、房间与 WebSocket 回归。

## 尚未实现

当前仍未实现投射物动画、单位类型、建筑升级、重新开局按钮、断线重连和长期对战的增量状态同步。单人关卡、AI、升级树与成就不属于当前“双人对战”目标。
