# 《绿色王国之战》局域网双人版 · 对比原版评估报告

> 评估日期：2026-07-20
> 评估对象：`D:\code\Green_king` 当前工作区（服务端权威 + 原生 Canvas 前端）
> 原版参照：IFGD《The Green Kingdom》/ 4399《绿色王国之战》，以及本项目 `docs/original_game_research.md` 的证据分级

## 一、项目现状总评

工程完成度相当高，已经超出"原型"范畴：

- **架构**：服务端权威（Node + Express + ws），100ms 固定步进战斗模拟；客户端只发意图、只做渲染。无客户端预测/回滚，局域网下完全够用且防作弊。
- **核心闭环完整**：中立岛屿 → 城堡初始底盘 → 建筑 8 邻格扩张/回退 → 树石 Clear → 矿山自动产金 → 兵营正交路线（最多 3 目标、绕障、Cancel）→ 出兵/近战/塔攻击 → 建筑摧毁 → 城堡被毁判胜/平局 → 重开。
- **联机体验**：创建/加入房间、6 位房号、断线自动重连（reconnect-banner）、双方确认重开（rematch-overlay）、移动端 Pointer Events 单指拖拽、7 张左右镜像对称地图。
- **素材**：接入了从原版 SWF 定向导出的绿/紫城堡、矿/塔/兵营、树石、士兵左右行军 PNG（仅本地非发布用途，已标注权利风险）。
- **质量**：`npm test` 当前 **53/53 全部通过**，覆盖地图、底盘、Clear、路线、战斗、胜负、房间隔离、WebSocket 回归等。

结论：对原版的"核心机制忠实度"和"工程完成度"都很高。下面逐项对比。

## 三、关键判断

1. **你的偏离几乎都是"刻意的 PvP 改造"或"主动简化"，而非疏漏。** 研究文档对每一条偏离都做了证据分级与边界说明，工程上非常干净。
2. **最值得补的是"局内升级树"**——它是原版"16 升级"在实时对战中的等价物，能显著提升策略深度，且架构（服务端权威）天然支持，路线图第 8 阶段已为此预留位置。
3. **音效与投射物动画是性价比最高的"体验补完"**，纯前端、零架构风险，建议下一步就做。
4. **点击回收金币不建议回退**，自动入账是当前原型最优解；如坚持还原原版，做成可选模式即可。
5. **单人 AI / 战役与当前"局域网双人"定位冲突**，若要追求"原版完整体验"，应作为独立分支而非并入本仓库。

## 四、推荐落地顺序

1. 音效 + 投射物/粒子动画（1 次小对话，前端）
2. 局内升级树（需动 `config` + `server.js` 模拟 + 前端面板，路线图第 8 阶段）
3. 本地成就/战绩（localStorage）
4. （可选）多兵种增强
5. （独立分支）单人 AI + 战役
