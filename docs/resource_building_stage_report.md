# 金币资源与固定节点建造阶段报告

目标：在已有权威地图和路线之上实现每房间独立金币、六个固定建筑位，以及矿山、塔、兵营建造。

`GAME_CONFIG` 集中定义：初始金币 400；矿山/塔/兵营成本 100/150/180；矿山每 2 秒收入 10。每个房间复制地图节点并维护独立 `playerState`、`nodes`、`routes`；矿山收入由服务端单一循环结算，空房被删除后不再参与结算。

客户端只发送 `{ type: "build", nodeId, buildingType }`。服务端验证房间满员、节点归属、建筑位为空、建筑类型合法和余额充足；成功后扣款、更新节点并广播 `game_state`。路线验证改为检查节点当前 `buildingType === "barracks"`，因此新兵营也可建立路线。

修改：`server/config.js`、`server.js`、`public/app.js`、`public/index.html`、`test/server.test.js`、`README.md`。自动测试覆盖金币初值、建筑位、扣款、重复和越权建造、矿山收入隔离、新兵营路线与既有路线测试。尚未进行本阶段真实电脑/手机联测；没有实现士兵、战斗、升级或胜负。下一阶段建议仅实现兵营服务端定时产兵与沿路线移动。
