# 局域网双人地图路线原型

本阶段提供：六位房间号、`player1`/`player2` 身份分配，以及服务端定义的左右对称固定地图。双方从各自兵营拖到相邻节点时，服务端验证并把正式路线同步给同房间两端。每个兵营只保留一条当前路线。

尚未实现：金矿、金币、建造、士兵、移动、战斗、防御塔、建筑生命、胜负、重连或公网联机。

## 启动

1. 安装 [Node.js LTS](https://nodejs.org/) 并在项目目录执行 `npm install`。
2. 双击 `start-server.cmd`，或运行 `npm start`。
3. 使用终端显示的 `LAN address(es)`（例如 `http://192.168.1.23:3000`）打开页面。服务监听 `0.0.0.0`，可供同一局域网访问。

## 电脑与手机联测

1. 电脑打开 LAN 地址并创建房间，记下六位房间号。
2. 手机连接同一 Wi-Fi，打开相同 LAN 地址后加入该房间。
3. 两端应显示相同地图；左方只能拖动 `player1` 兵营，右方只能拖动 `player2` 兵营。
4. 按住自己的高亮兵营，拖到直接相连的中立节点或敌方建筑后松开。拖动时只显示本机临时线；收到服务端确认后，两端显示正式路线。

Windows Defender 弹窗出现时，仅允许 Node.js 在“专用网络”通信；不要自动修改防火墙。公共 Wi-Fi 可能启用客户端隔离，不能用于联测。
