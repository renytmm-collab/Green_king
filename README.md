# 《绿色王国之战》机制启发的局域网双人原型

这是 4399《绿色王国之战 / The Green Kingdom》核心建造、行军与战斗机制的局域网双人变体。房间经济、单位移动、伤害和胜负全部由服务端权威模拟。

## 当前玩法

- 服务端为每个房间创建独立的 9×18 不规则岛屿。
- 双方城堡开局只占有周围一圈底盘，其余可通行土地为中立地。
- 玩家只能在自己的空底盘建造矿山、防御塔或兵营。
- 每座完成的建筑会把周围八格铺成己方底盘。扩张可以覆盖中立地和对方空底盘，但不会直接覆盖对方建筑或城堡所在格。
- 树木和岩石被底盘扩张覆盖后可以 `Clear`；清理后该格才可建造。树木花费 40、岩石花费 60。
- 每座矿山每 2 秒自动增加 10 金币，不需要点击拾取。
- 从己方兵营拖过最多三个敌方建筑，并在最后目标上松开。路线吸附到方格中心，只能水平或垂直前进；跨行、跨列拖动时会生成 90° 转角。
- 每段正交折线都不能穿过悬崖、树木、岩石或非目标建筑，因此可以主动转折绕开障碍；不同兵营的路线可以交叉。
- 同一兵营重画会覆盖旧路线；短按已有路线的兵营可显示 `Cancel`。
- 兵营每 2.4 秒沿正式路线生成一名士兵；士兵会逐个攻击路线目标，折返时切换左右朝向，并与近距离敌军交战。
- 防御塔自动攻击范围内最近的敌军。建筑和士兵都有生命值。
- 建筑摧毁后，由剩余建筑重新计算底盘；以被摧毁建筑为起点或目标的正式路线会取消。
- 摧毁敌方城堡即可获胜；双方城堡在同一个服务端时间步归零则判为平局。

## 经济

- 初始金币：400
- 矿山：100
- 防御塔：150
- 兵营：180
- 树木清理：40
- 岩石清理：60
- 矿山收入：每 2 秒自动到账 10

身份、建造、底盘归属、Clear、余额、目标顺序和路线碰撞均由服务端验证。客户端只上传拖拽采样点，方格中心吸附、水平/垂直折点、建筑中心检查点和合法性均由服务端生成与判定。

## 启动与局域网联测

首次运行：

```powershell
npm install
```

启动：

```powershell
npm start
```

也可以双击 `start-server.cmd`。电脑浏览器打开终端显示的地址并创建房间，把 LAN 地址和六位房间号提供给同一 Wi-Fi 下的手机浏览器加入。

人工联测建议：

1. 确认双方开局只拥有城堡周围一圈底盘。
2. 在边缘建造，确认新建筑会铺开周围八格并允许继续推进。
3. 让一方扩张到另一方的空底盘，确认空地变色，但对方建筑格不被覆盖。
4. 扩张到树木后执行 `Clear`，再在释放的格子建造。
5. 建造矿山并等待 2 秒，确认余额自动增加且画面不出现待拾取金币。
6. 从兵营依次拖过一至三个敌方建筑，沿方格横向或纵向转弯绕过树木或石头，确认正式路线吸附到格心并只形成 90° 折点。
7. 验证穿过障碍、悬崖和非目标建筑的线段会被拒绝，并验证 `Cancel`。
8. 等待兵营出兵，确认士兵沿目标顺序移动、交战并攻击建筑。
9. 确认防御塔自动开火、建筑摧毁后底盘回退，城堡摧毁后显示胜负。
10. 在手机上检查单指拖拽、横竖屏和窗口缩放后的命中位置。

## 代码结构

- `server.js`：房间、权威状态、建造、底盘重算、经济、路线和固定步进战斗模拟。
- `server/config.js`：经济、地图、城堡、障碍、建筑生命值与战斗数值。
- `public/app.js`：WebSocket、菜单与 Pointer Events。
- `public/renderer.js`：坐标转换、命中与 Canvas 绘制。
- `public/visual-config.js`：视觉参数、原版 PNG 映射与动画元数据。

当前本地版本使用从原版 SWF 提取的绿色/紫色城堡、矿山、防御塔、兵营、树木、岩石与士兵行军帧；客户端连续播放单位动画，并按对象底部位置处理跨格遮挡。地形、路线、界面和加载失败时的几何 fallback 仍由本项目绘制。素材来源与角色映射见 `public/assets/original/README.md`。
